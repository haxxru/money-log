# 머니로그 기획서

## 1. 제품 개요

- 제품명: 머니로그
- 목적: 개인 사용자가 월 단위로 수입/지출/예산/목표를 기록하고, 대시보드에서 재무 상태를 빠르게 파악하도록 지원한다.
- 핵심 가치:
  - 입력은 빠르게, 분석은 직관적으로
  - 월 단위 소비 통제(예산)와 중장기 축적(목표)을 동시에 관리
  - 가입 직후 초기 잔액 기반으로 즉시 사용 가능, 급여 설정은 별도 설정 화면에서 관리

---

## 1.1 기술 스택

- 프론트엔드: Next.js 14 (App Router), React 18, TypeScript
- UI: Tailwind CSS, shadcn/ui 계열 컴포넌트, Recharts
- 백엔드/BaaS: Supabase (Auth, Postgres, RLS)
- 배포: Vercel

---

## 2. 타깃 사용자

- 1차 타깃:
  - 월급 기반 개인 사용자
  - 가계부를 단순 입력이 아닌 “한 화면 요약”으로 보고 싶은 사용자
- 주요 페인포인트:
  - 어디에 얼마나 쓰는지 월말에야 알게 됨
  - 예산 초과를 사후에 인지함
  - 목표 저축 진행률을 수기로 계산함

---

## 3. 문제 정의

- 거래 기록은 하더라도 월별 흐름(수입/지출 추세, 카테고리 비중)을 한눈에 보기가 어렵다.
- 예산과 거래가 분리되면 “실제 소진률” 확인이 느리고 부정확해진다.
- 가입 직후 기본 데이터가 비어 있으면 제품 정착률이 떨어진다.

---

## 4. 목표 및 성공 지표

## 제품 목표

- 거래 입력부터 분석 확인까지 1분 내 완료 가능한 UX 제공
- 월별 지출 총액, 카테고리 비중, 예산 소진률을 실시간에 가깝게 반영
- 가입 즉시 사용 가능한 초기 상태(닉네임/초기 잔액/월급 정보) 확보
  - 가입 즉시 사용 가능한 초기 상태(닉네임/초기 잔액) 확보
  - 급여 정보는 설정 탭에서 관리 가능한 UX 제공

## 성공 지표 (초기 운영 기준)

- 가입 후 첫 거래 입력 완료율
- 월간 활성 사용자당 평균 거래 입력 건수
- 예산 설정 사용자 비율
- 목표 생성 사용자 비율
- 대시보드 재방문율

---

## 5. 핵심 기능 범위

## 5.1 인증/회원

- Supabase Auth 이메일/비밀번호 가입/로그인/로그아웃
- 가입 입력:
  - 닉네임
  - 초기 잔액
- 가입 시 `auth.users.raw_user_meta_data` 저장 후 DB 트리거로 `user_profiles` 초기화
- 급여 설정:
  - 설정 탭(`/settings`)에서 월급일/월급액 저장

## 5.2 대시보드

- 월 기준 인사/요약 헤더
- KPI 카드:
  - 잔액
  - 이번 달 수입
  - 이번 달 지출
  - 저축
- 차트:
  - 현금 흐름(기간 선택: 1주/1개월/3개월/1년)
  - 카테고리별 지출 비중(도넛)
- 거래 내역 위젯:
  - 검색/카테고리 필터
  - 거래 추가 다이얼로그(DB 저장)
  - 거래 수정/삭제 다이얼로그(DB 반영)
- 예산 현황 위젯(카테고리별 소진률)

## 5.3 거래 내역

- 거래 추가
- 거래 목록 조회
- 행 단위 수정
- 행 단위 삭제
- 유형(수입/지출), 카테고리, 금액, 날짜, 내용 관리

## 5.4 예산

- 카테고리별 월 예산 저장 (upsert)
- 월 사용액/소진률 계산 및 시각화

## 5.5 분석

- 최근 30일 지출 추세
- 카테고리 비중 분석

## 5.6 목표

- 목표 추가
- 현재 진행 금액 수정
- 목표 삭제

## 5.7 월급 자동 반영

- 로그인 시 이번 달 월급 거래 누락 시 자동 생성
- 예외 규칙:
  - 가입한 달에는 `salary_day < 가입일`이면 자동 생성하지 않음

## 6. UX/디자인 원칙

- `ui-demo` 전역 디자인 시스템을 기준으로 통일
- 다크/라이트 모드 지원
- 정보 밀도 높은 대시보드 구조 유지
- 주요 액션(거래 추가/수정/삭제)은 뷰 전환 없이 처리

---

## 7. 데이터/도메인 설계 요약

## 주요 테이블

- `expenses`
  - 거래 원장
  - `amount`는 양수 저장, 의미는 `tx_type`으로 구분
  - `tx_type`: `income` | `expense`
  - `category`: 식비/주거/교통/쇼핑/여가/수입/기타
- `user_profiles`
  - 사용자 재무 초기값/월급 설정
- `budgets`
  - 카테고리별 월 예산
- `goals`
  - 목표 금액/진행 금액/만기일

## 보안/권한

- RLS 기반 사용자 소유 데이터 분리 (`user_id = auth.uid()`)
- 서버 액션에서 인증 사용자 검증 후 쓰기 작업 수행

---

## 8. 사용자 시나리오

## 시나리오 A: 신규 사용자 온보딩

1. 회원가입 시 닉네임/초기 잔액 입력
2. 가입 완료 후 로그인
3. 대시보드에서 초기 상태 확인
4. 설정 탭에서 정기 급여 입력
5. 첫 거래 추가

## 시나리오 B: 월간 소비 관리

1. 거래를 지속 입력
2. 대시보드에서 월 지출/카테고리 비중 확인
3. 예산 탭에서 카테고리별 한도 조정
4. 예산 소진률 기반 소비 조정

## 시나리오 C: 목표 저축 관리

1. 목표 생성
2. 월 단위로 진행 금액 업데이트
3. 목표 달성률 추적
